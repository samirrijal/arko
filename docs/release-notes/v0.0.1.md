# arko-engine v0.0.1 — first public cut

The first tagged release of the Arko LCA calculation engine. This is a
Phase 0 milestone: the engine is a public, reproducible artifact, but the
surface area is deliberately small. Feature breadth lands across Phase 1
(weeks 3–10).

Tagged: 2026-04-19 · commit [`daa91c4`](https://github.com/samirrijal/arko/commit/daa91c4)

## What's included

- **Spec v0.1 (`arko-calc`)** — frozen mathematical specification at
  [`specs/calc/v0.1.md`](../../specs/calc/v0.1.md). The engine is linked
  to this spec version; future spec revisions will ship as v0.2, v0.3.
- **13-crate Rust workspace** covering the full LCA pipeline:
  `core`, `units`, `methods`, `solvers-dense`, `solvers-sparse`,
  `parameters`, `uncertainty`, `sensitivity`, `license`, `differential`,
  `validation`, `io-ecospold2`, `io-ilcd`.
- **Solvers**: dense LU via `nalgebra`, sparse LU via `faer` 0.20.
- **Two method presets**: IPCC AR5 GWP100 (WG1 Ch8 T8.A.1) and IPCC
  AR6 GWP100 (WG1 Ch7 T7.15), the latter with CH4 fossil / non-fossil
  origin splitting.
- **Determinism contract** (spec §7.1): BLAKE3 canonical hash over studies,
  seeded Monte Carlo with bit-identical reproducibility.
- **Conformance framework** (`differential` crate): `TestVector`,
  `ConformanceLevel` (L1/L2/L3), tolerance classes (`ReferenceParity`,
  `CrossImpl`), and a runner that produces `ConformanceReport`.
- **Units DSL** (`units` crate): dimension algebra, SI prefixes,
  canonical form, roundtripping.
- **Reader skeletons**: `io-ecospold2` (single-file) and `io-ilcd`
  (single `processDataSet`). Bundle linking lands in Phase 1.
- **247 tests green** on Linux, macOS, and Windows (Rust 1.95).

## What's not yet included

- ILCD multi-file bundle linking (Phase 1 weeks 3–4)
- EPDX and OpenLCA JSON-LD importers (Phase 1 weeks 5–6)
- ReCiPe 2016 Midpoint, EF 3.1, CML 2001 method presets (Phase 1 weeks 7–8)
- `FactoredSolver` trait for amortized LU across sensitivity iterations
- Any UI — engine-only for Phases 0 and 1

## How to build

Requires Rust 1.95 (pinned via `rust-toolchain.toml`).

```
git clone https://github.com/samirrijal/arko.git
cd arko/engine
cargo build --workspace
cargo test --workspace
```

## How to verify

Run the integration test for the worked example from spec §16:

```
cargo test -p arko-core --test minimal_example
```

This reproduces `s = [1.0, 0.5]`, `g = [2.05]`, `h = [2.05]` from the
two-process worked example under reference-parity tolerance (ε_abs =
1e-12).
