## Abstract

This specification defines the **license-tier policy language** of an
Arko-conforming engine: how per-process license restrictions propagate
through a calculation, how derivative-rule triggers fire, how the
combined `Authorization` decision for a publish/share/export attempt
is computed, and what an Arko-conforming engine **MUST**, **SHOULD**,
or **MAY** record in its audit log.

It does **not** redefine the storage types (those are normative in
`specs/calc/v0.1.md` §11). It defines the *semantics* layered on top
of those types — the rules an implementation must follow when a user
asks "may I publish this result?".

This document is language- and implementation-agnostic. Any
implementation that conforms to this specification and produces the
same `Authorization` for the same `(study, computed, intent, now)`
tuple as the reference implementation is a **conforming Arko license
evaluator**.

1. The reference presets (§9) have been verified against the actual

2. At least one independent implementation passes the conformance

## 2. Conventions

- **MUST**, **MUST NOT**, **SHOULD**, **SHOULD NOT**, **MAY** are used
  per [RFC 2119] and [RFC 8174].
- All identifiers in code fragments are case-sensitive.
- Type fragments (`Authorization = { … }`) follow the same
  abstract-syntax style as `specs/calc/v0.1.md`. They are normative;
  any concrete serialization (JSON, MessagePack, Cap'n Proto) **MUST**
  preserve the field set, the field names, and the value enumerations.
- All timestamps are UTC, RFC 3339-formatted.
- Indices are 0-based.

## 3. Guiding principle

**The solve is always legal.**

License enforcement gates *publication, sharing, and export* of
results. It does **not** gate the calculation itself. An engine
**MUST NOT** refuse to compute a study because of license
considerations — the matrices are private mathematical objects until
the operator chooses to act on them.

This separation places enforcement at the natural business-logic
boundary (the publish endpoint) and keeps the solver free of
license-aware code paths. It also lets internal auditing,
debugging, and reproducibility checks run without re-consent.

Two consequences:

1. The license evaluator **MUST** be invokable independently of the
   solver. It takes the *output* of a solve plus an `Intent` and
   returns an `Authorization`. The solver and the evaluator share
   no state beyond what is captured in `Provenance`.
2. The evaluator **MUST** be deterministic for fixed inputs. Same
   `(study, computed, intent, now)` tuple → same `Authorization`.

## 4. Intent enumeration

The `Intent` enumeration declares what the operator wants to do with
a result. Exactly one variant is in scope per `authorize` call.

```
Intent = "publish" | "share" | "export"
```

| Variant   | Meaning                                                    |
| --------- | ---------------------------------------------------------- |
| `publish` | External publication: report, EPD, marketing material.     |
| `share`   | Sharing the inventory itself to another workspace.         |
| `export`  | Exporting raw data as an open format (ecospold2, EPDX, …). |

An engine **MAY** add further intents in a v0.2+ revision (e.g.
`internal_review`, `regulatory_submission`); each new intent **MUST**
have its own base flag on `LicenseTier`.

## 5. Authorization decision

The `Authorization` produced by `authorize(intent, study, computed, now)`
is the combination of three signals, processed in the order below.
The combination rule is **most severe wins** for `Outcome`, with the
side-effect lists (`fired`, `blocking_sources`, `expired_sources`)
populated as documented for each signal.

```
Authorization = {
    intent:            Intent,
    outcome:           Outcome,
    fired:             [FiredRule],
    blocking_sources:  [String],
    expired_sources:   [String],
}

Outcome = "allowed" | "warn" | "watermark" | "blocked"
```

`Outcome` carries a strict severity ordering:

```
allowed < warn < watermark < blocked
```

The combined outcome is the maximum (most severe) over all three
signals. An engine **MUST** preserve this ordering when serializing or
deserializing `Outcome`.

### 5.1 Signal 1 — Base flag

The `EffectiveRestriction` on `Computed.provenance.restrictions`
already encodes the §11.2 boolean-AND join of every contributing
tier's `allow_*` flags. Map the intent to its corresponding base
flag:

| Intent    | Base flag        |
| --------- | ---------------- |
| `publish` | `allow_publish`  |
| `share`   | `allow_share`    |
| `export`  | `allow_export`   |

If the base flag is `false`, the outcome **MUST** be `blocked` and
every entry of `effective_restriction.sources` **MUST** be appended
to `blocking_sources` in source order, de-duplicated.

A base-flag block is **unconditional**: no derivative rule firing,
no expiry refresh, and no caller can upgrade it. The other two
signals are still evaluated (so callers see the full picture in
`fired` and `expired_sources`), but the outcome stays `blocked`.

### 5.2 Signal 2 — Derivative rules

Every contributing process's `derivative_rules` are evaluated against
the scaling vector `s = computed.scaling`. A process `j` is
*contributing* iff `|s[j]| > ε_presence` where
`ε_presence = 1e-9` (matching §11.2 of the calc spec).

For each contributing `j`, the engine **MUST** look up the tier
`study.license_tiers[study.processes[j].license_tier]` and evaluate
each `DerivativeRule` of that tier *in declaration order* against
`s[j]`. See §6 for trigger semantics.

For every fired rule the engine **MUST** append a record to
`fired`:

```
FiredRule = {
    process_index:   u32,             // index into Study.processes
    tier_source:     String,          // tier.source
    action:          DerivativeAction,
    message:         String,          // verbatim from rule.message
    scaling_factor:  f64,             // the s[j] that caused the trigger
}
```

The combined outcome is updated to the maximum of the current
outcome and the rule's action (mapped via §7).

If the rule's action is `block`, the engine **MUST** append
`tier.source` to `blocking_sources` (de-duplicated, insertion-ordered).

The `fired` list is returned **verbatim** even when the outcome is
`blocked`. The audit-log obligation of §8 requires every fired rule
to be visible to the operator.

### 5.3 Signal 3 — Expiry

For every contributing process `j` (same `ε_presence` threshold), if
its tier carries an `expiry` and `expiry < now`, the engine **MUST**:

1. Set the outcome to `blocked`.
2. Append `tier.source` to `expired_sources` (de-duplicated,
   insertion-ordered).
3. Append `tier.source` to `blocking_sources` (de-duplicated,
   insertion-ordered).

`now` is **caller-provided**, not read from the system clock. This
makes the evaluator deterministic and lets a result's
`provenance.computed_at` drive replay-time authorization for audit
purposes.

An engine **MAY** offer a convenience wrapper that defaults `now` to
`Utc::now()`, but the underlying evaluator **MUST** accept an
explicit `now`.

## 6. Derivative-rule trigger semantics

```
DerivativeRule = {
    when:    DerivativeTrigger,
    action:  DerivativeAction,
    message: String,
}

DerivativeTrigger = ScalingGe { threshold: f64 }
                  | Always
```

### 6.1 `ScalingGe { threshold }`

The trigger fires for process `j` iff `|s[j]| ≥ threshold`. The
absolute value is mandatory: a process with `s[j] = -0.6` and a
threshold of `0.5` **MUST** fire. This handles the producer-negative
sign convention (§4.4 of calc) and substitution-credited co-products
(§6.5) symmetrically.

The threshold is compared with `≥`, not `>`. A process with
`s[j] = 0.5` and threshold `0.5` **MUST** fire. This is the
"contributes ≥ 1%" reading that ecoinvent's attribution clause
relies on, where 1% of a unit functional flow is exactly `0.01`.

`threshold` **MUST** be a finite, non-negative `f64`. Behavior on
`NaN` or negative thresholds is implementation-defined; an engine
**SHOULD** reject such rules at study-validation time.

### 6.2 `Always`

The trigger fires unconditionally for every contributing process
(`|s[j]| > ε_presence`). This is the standard pattern for vendor
licenses where any presence at all triggers a re-licensing
requirement (e.g. the `sphera_strict` preset).

### 6.3 Future triggers (v0.2+)

The trigger enumeration is **open for extension**. Future versions
**MAY** add:

- `ImpactGe { category, threshold }` — fire when the result of a
  specific impact category exceeds a threshold.
- `MarketShareGe { product, threshold }` — fire when the
  characterised result attributes ≥ X% of total market.
- `Geographic { regions: [String] }` — fire when the contributing
  process is sourced from one of the listed regions.

A v0.1 evaluator **MUST** reject unknown trigger variants with a
clear error rather than silently treat them as `Always` or a no-op.

---

## 7. Action → Outcome mapping

```
DerivativeAction = "block" | "warn" | "watermark"
```

| `DerivativeAction` | `Outcome`   |
| ------------------ | ----------- |
| `warn`             | `warn`      |
| `watermark`        | `watermark` |
| `block`            | `blocked`   |

The mapping is total, fixed across versions, and bidirectional in
severity. An engine **MUST NOT** invent additional actions in v0.1.

## 8. Audit-log requirements

When `intent ∈ {publish, share, export}` is acted upon by the
caller (i.e., the operator proceeds with publishing), the engine
**MUST** record an audit entry containing **at minimum**:

```
AuditEntry = {
    timestamp:           DateTime,        // wall clock at publish time
    operator:            String,          // matches Provenance.computed_by
    study_hash:          String,          // matches Provenance.study_hash
    spec_version:        String,          // license spec version
    engine_version:      String,
    intent:              Intent,
    outcome:             Outcome,
    fired:               [FiredRule],     // verbatim from Authorization
    blocking_sources:    [String],
    expired_sources:     [String],
    operator_attested:   bool,            // true iff operator clicked "I accept"
                                          // for warn / watermark outcomes
}
```

The audit entry **MUST** be persisted before the publish action
completes. An engine **MUST NOT** allow a `blocked` outcome to be
overridden by the operator at this layer; overrides require an
out-of-band re-consent flow that produces a fresh
`LicenseTier.expiry` extension or a new tier altogether.

For `warn` and `watermark` outcomes the engine **MUST** require
explicit operator acknowledgement and record `operator_attested =
true`. For `allowed` outcomes the field **MAY** be omitted or set
to `false`.

The audit log **SHOULD** be append-only. An engine **MUST NOT**
expose a "redact" or "delete" operation at the API layer; correction
is by appending a counter-entry that references the original by
timestamp and study hash.

## 9. Standard library presets

The reference implementation ships preset constructors in
`arko_license::presets`. These presets are **conservative defaults**
derived from publicly-summarized license terms. They are *not* legal
advice and do *not* reproduce any vendor's EULA. Operators **MUST**
verify every preset against their signed license agreement before
relying on it for a specific deployment.

### 9.1 `ecoinvent_v3(version)`

```
LicenseTier {
    source: "ecoinvent-{version}",
    allow_publish: true,
    allow_share:   false,
    allow_export:  false,
    derivative_rules: [
        DerivativeRule {
            when: ScalingGe { threshold: 0.01 },
            action: warn,
            message: "Result includes material contribution from
                      ecoinvent-{version}; attribution required per
                      EULA, derivative sharing restricted",
        },
    ],
    expiry: None,
}
```

Rationale: the ecoinvent EULA permits external publication of
*derived results* but not redistribution of inventory or raw data.
The 1% threshold is the standard attribution-significance line.

### 9.2 `sphera_strict(version)`

```
LicenseTier {
    source: "sphera-{version}",
    allow_publish: false,
    allow_share:   false,
    allow_export:  false,
    derivative_rules: [
        DerivativeRule {
            when: Always,
            action: block,
            message: "Sphera / GaBi dataset version {version} —
                      external publication requires re-licensing
                      with the vendor",
        },
    ],
    expiry: None,
}
```

Rationale: the standard Sphera commercial license requires per-use
re-licensing for external publication; the safest default is to
block and force the operator through the vendor's process.

### 9.3 `open_cc_by(source)`

```
LicenseTier {
    source: source,
    allow_publish: true,
    allow_share:   true,
    allow_export:  true,
    derivative_rules: [
        DerivativeRule {
            when: ScalingGe { threshold: 0.01 },
            action: warn,
            message: "Attribution required (CC-BY 4.0): {source}",
        },
    ],
    expiry: None,
}
```

Rationale: CC-BY-style open licenses permit all three intents but
require attribution. The 1% threshold matches the ecoinvent
significance line for cross-database consistency.

### 9.4 `custom_user()`

```
LicenseTier::permissive("custom-user")
```

Fully permissive, no rules, no expiry. For operator-authored
processes that carry no third-party licensing obligations.

### 9.5 Adding new presets

A v0.1 implementation **MAY** add additional presets (e.g.
`ecoinvent_apos`, `agrybalyse`, `worldsteel`, `idemat`). Each new
preset **MUST**:

- Carry a clear `source` string of the form
  `"<provider>-<version>"`.
- Document the EULA clauses each base flag and rule corresponds to.
- Include the §9 disclaimer in its rustdoc.

## 10. Conformance

A conforming Arko license evaluator **MUST** produce an
`Authorization` for any `(study, computed, intent, now)` input that
is bit-identical (in serialized form) to the reference
implementation's, modulo:

- Whitespace and field ordering in the JSON serialization.
- Equivalent representations of timestamps (RFC 3339-compliant).

The conformance corpus (to be published alongside this spec) will
contain:

- Every preset of §9 evaluated against single-process and
  multi-process studies, at scaling factors that bracket every
  threshold edge.
- Mixed-tier studies that exercise the §11.2 join (any-restricts-the-
  result semantics).
- Expired-tier studies at `now` values that bracket the expiry
  instant.
- Multi-rule tier studies that exercise the most-severe-wins
  combination across `warn`, `watermark`, `block`.

Until the conformance corpus is published, the reference
implementation in `engine/license/` is itself the conformance target
and its test suite is the de-facto corpus.

## 11. Determinism contract

The license evaluator inherits §7.1 of the calc spec. In particular:

- The `fired` list **MUST** be byte-stable for a fixed
  `(study, computed)` pair. Firing order is the natural one:
  outer loop over processes in index order, inner loop over the
  tier's `derivative_rules` in declaration order.
- `blocking_sources` and `expired_sources` are insertion-ordered and
  de-duplicated; the de-duplication step **MUST NOT** sort the list
  (sorting would hide ordering bugs).
- Floating-point comparisons use the canonical IEEE 754 semantics
  for `f64`. `NaN` scaling factors **MUST NOT** trigger any
  `ScalingGe` rule, but **SHOULD** surface as a study-validation
  warning at compute time (`E_PARAM_NONFINITE` if from a parameter,
  or `E_INTERNAL` otherwise — `NaN` should never reach the
  evaluator from a successful solve).

---

## 12. Error handling

The evaluator itself does not return errors at v0.1 — every input
that satisfies the calc spec produces an `Authorization`. Failure
modes are pushed to surrounding layers:

- Malformed `LicenseTier` (e.g. negative `ScalingGe` threshold) is
  caught at study validation (`arko-validation`).
- Out-of-range `license_tier` index in `ProcessMeta` is caught at
  study validation (`E_INTERNAL` per calc §13.1).
- Non-finite scaling factors **MUST** be caught at solve time, not
  at evaluation time. An evaluator that receives non-finite
  `s[j]` **MAY** treat it as zero (no contribution) but **SHOULD**
  surface a `W_INTERNAL` warning to the caller.

---

## 13. Forward compatibility

The on-disk serialization of `LicenseTier`, `DerivativeRule`,
`Authorization`, and `FiredRule` is governed by the calc spec's
§7.1 determinism contract: field ordering follows the abstract-syntax
order in this document, and unknown fields **MUST** be preserved
on round-trip.

A v0.2 evaluator **SHOULD** accept v0.1 tiers and rules unchanged.
A v0.1 evaluator that encounters a v0.2 trigger variant or a new
intent **MUST** reject the input with a clear error rather than
silently downgrade to a `block` or `allowed` decision.

## Appendix B — Out of scope (v0.1)

The following will be addressed in v0.2 or later:

- **Cryptographically-signed tiers.** A `LicenseTier.signature`
  field that lets a vendor sign the `(source, allow_*, rules,
  expiry)` tuple, so an audit reviewer can verify the tier was not
  tampered with after issuance.
- **Per-impact-category restrictions.** A vendor that licenses
  climate-change results but not toxicity results.
- **Time-windowed restrictions.** A tier that becomes restrictive
  on a future date (the inverse of `expiry`).
- **Operator-attestation persistence.** A standardized format for
  the "I accept" record that survives across deployments.
- **Federated audit logs.** Cross-organization audit replication
  with conflict-free merging.
